# Markdown 编辑器行为规范（AI Coding 版）

本文档定义源码编辑器中的 Markdown 编辑行为预期。

目标对齐基线：

* Obsidian（Source / Live Preview）
* Typora（块级编辑）

---

## 术语定义（新增）

* 列表 marker：

  * 无序：`-` / `*` / `+`
  * 有序：`1.` / `2.` 等
* 任务列表：`- [ ]` / `- [x]`
* 引用标记：`>`
* 缩进：4 个空格
* 当前行：光标所在行
* 空列表项：仅包含 marker + 空格

---

# 核心规则

---

## Enter

### 规则 1：非空列表项续写

**条件：**

* 当前行是列表项
* marker 后存在非空内容

**行为：**

* 在下一行生成同类型列表项

**示例：**

```md
- item|
→ Enter →
- item
- |
```

---

### 规则 2：有序列表递增

**条件：**

* 当前行为有序列表

**行为：**

* 下一行编号 = 当前编号 + 1

**示例：**

```md
9. item|
→ Enter →
9. item
10. |
```

---

### 规则 3：空列表项退出

**条件：**

* 当前行是空列表项

**行为：**

* 删除当前 marker
* 若有父级 → 回退一级
* 若无父级 → 转为普通段落

**示例：**

```md
- |
→ Enter →
|
```

嵌套：

```md
- a
    - |
→ Enter →
- a
|
```

---

### 规则 4：任务列表续写

**条件：**

* 当前行为任务列表

**行为：**

* 保持 `[ ]` 或 `[x]` 状态

**示例：**

```md
- [x] done|
→ Enter →
- [x] done
- [ ] |
```

---

### 规则 5：引用块续写

**条件：**

* 当前行以 `>` 开头

**行为：**

* 下一行保持相同数量的 `>`

**示例：**

```md
> text|
→ Enter →
> text
> |
```

---

### 规则 6：空引用退出

**条件：**

* 当前行仅包含 `>`（或 `> `）

**行为：**

* Enter 后退出引用

---

### 规则 7：代码块内 Enter

**条件：**

* 光标在 ``` 包裹的代码块内

**行为：**

* 仅换行，不生成任何结构

---

### 规则 8：标题 Enter

**条件：**

* 当前行为标题（`#` 开头）

**行为：**

* Enter 后生成普通段落

---

### 规则 9：混合结构续写

**示例：**

```md
> - item|
→ Enter →
> - item
> - |
```

---

## Backspace

### 规则 1：删除结构标记

**条件：**

* 光标紧跟 marker 后

**行为：**

* 删除一层结构

**示例：**

```md
- |text
→ Backspace →
|text
```

---

### 规则 2：逐层回退

```md
> > text|
→ Backspace →
> text|
→ Backspace →
text|
```

---

### 规则 3：列表降级

```md
    - item|
→ Backspace →
- item|
→ Backspace →
item|
```

---

### 规则 4：内容保护

* 不允许一次删除整行内容
* 仅删除结构前缀

---

## Tab / Shift-Tab

### 规则 1：列表缩进

**行为：**

* Tab → 增加一层缩进（4 空格）

```md
- item
→ Tab →
    - item
```

---

### 规则 2：列表回退

```md
    - item
→ Shift-Tab →
- item
```

---

### 规则 3：多行选区

**行为：**

* 对所有选中行生效

---

### 规则 4：编号重排

```md
1. a
    1. b
→ Shift-Tab →
1. a
2. b
```

---

### 规则 5：非列表行

* Tab → 插入 4 空格
* Shift-Tab → 删除最多 4 空格

---

### 规则 6：光标保持

* 尽量保持原列位置

---

## 有序列表

### 规则 1：同层递增

* 同层级自动递增
* 子层级重新从 1 开始

---

### 规则 2：模式（重要）

支持两种模式：

#### 严格模式（Typora）

* 自动修复编号连续性

#### 宽松模式（Obsidian）

* 保留原编号

---

### 规则 3：删除后行为

```md
1. a
2. b
3. c
```

删除 `2. b`：

* 严格：1,2
* 宽松：1,3

---

## 列表切换命令

### Mod-Shift-L（无序列表）

**行为：**

* 对每一行添加/移除无序 marker

```md
text → - text → text
```

---

### Mod-Shift-O（有序列表）

```md
a
b
c
→
1. a
2. b
3. c
```

---

### 保留缩进

```md
    text →     - text
```

---

## 引用 / 标题命令

### Mod-Shift-.（引用）

```md
text → > text → text
```

---

### 引用层级

```md
> text → >> text → text
```

---

### Mod-Shift-H（标题）

循环：

```md
text
→ # text
→ ## text
→ ...
→ text
```

---

## 行内格式命令

### Mod-B（粗体）

```md
text → **text**
**text** → text
```

---

### Mod-I（斜体）

```md
text → *text*
```

---

### Mod-`（行内代码）

```md
text → `text`
```

---

### Mod-K（链接）

```md
text → [text](url)
```

无选区：

```md
[]()
```

---

### 光标行为

```md
| → Mod-B → **|**
```

---

## 粘贴行为（新增）

### 规则 1：列表内粘贴

```md
- |
粘贴：
a
b
```

结果：

```md
- a
- b
```

---

### 规则 2：自动识别

* URL → 自动转链接（可配置）
* 富文本 → 转 Markdown

---

## 撤销 / 重做（新增）

### 规则

* 单次结构操作 = 单个 undo step
* 不拆分为多个步骤

---

## 空行行为（新增）

### 规则

* 连续 Enter 两次 → 退出当前结构

---

## 回归检查清单（增强）

* 无序列表缩进/回退正确
* 有序列表编号递增正确
* `10.` → `11.` 正确
* 任务列表续写正确
* 空列表退出正确
* 多行 Tab 生效
* 多行 Shift-Tab 生效
* 引用层级正确
* 标题切换不破坏缩进
* 粘贴行为正确
* 混合结构（引用+列表）正确
* undo/redo 正确

## 实现建议（给 AI coding 用）

### 推荐数据结构

```ts
type Block =
  | Paragraph
  | ListItem
  | OrderedListItem
  | TaskItem
  | Blockquote
  | Heading
  | CodeBlock
```

---

### 输入处理模型

```text
KeyEvent
→ 判断上下文（AST / 行结构）
→ 应用规则（Enter / Backspace / Tab）
→ 生成新结构
→ 渲染
```

---

### 优先级（重要）

1. 代码块
2. 行内结构
3. 列表 / 引用
4. 普通文本
